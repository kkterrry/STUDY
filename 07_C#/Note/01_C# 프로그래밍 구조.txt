프로젝트
-> 하나이상의 cs파일로 구성
-> Main (o) -> EXE
    Main (x) -> DLL
-> EXE와 DLL을 어셈블리라고 칭함

클래스와 인스턴스
클래스 -> 실세계에 존재하는 사물 또는 개념을 모델링 한것
인스턴스 -> 클래스 타입으로 사용되어 프로그램에서 실행할 수 있도록 준비된 상태

클래스의 선언
접근제한자 class 클래스명 {
	속성정의;
	행위정의;
}
클래스는 하나의 .cs 파일에 하나 이상 선언할 수 있음
-> 하나의 cs파일에 하나의 클래스 정의 하는것이 유지보수 고려시 좋음

메서드의 역할
-> 모든 실행문은 메서드에서만 실행가능
-> 객체의 로직을 명령문으로 구현
-> 함수, 서브루틴, 프로지져라고도 불림
-> 메서드는 다른 메서드에서 호출되는 형식으로 사용됨
-> 재사용가능
-> 다른클래스 있는경우 호출 : 클래스.인스턴스변수.메서드명(파라미터)
-> 같은클래스에 있는 메서드 호출 : 메서드명(파라미터)
