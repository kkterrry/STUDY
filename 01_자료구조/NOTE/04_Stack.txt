- Stack
	- 쌓아 올린 건초더미
	- 한쪽 입구를 통해서 입력(push)을 하고 삭제(pop)를 하는 구조
	- 가장 나중에 들어온 데이터가 가장 먼저 나온다(LIFO : Last In First Out)-> 후입선출구조
	- 데이터가 꽉찬 경우 : stack overflow
	- 데이터가 없는 경우 : stack underflow
	- 구조
		1. top : 가장 나중에 들어온 데이터를 가리키는 역할
		- 스택은 top을 통해서 데이터가 입력되거나 삭제된다
		2. push : 데이터 입력
		3. pop : 데이터 삭제
		4. peek : 가장 위에 있는 데이터 출력

	- 구현
		1. 순차리스트 : 배열
		2. 연결리스트 : 개수의 제한 없음
